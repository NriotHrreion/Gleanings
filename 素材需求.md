## 1. 命名规范与目录结构

```
assets/
├─ bg/        场景背景     BG_<scene>.png
├─ char/      人物立绘     CH_<name>[_<state>][_<expr>].png
├─ items/     信物/徽章    IT_<type>_<name>[_<variant>].png
├─ cards/     文化卡        CARD_<slug>.png / CARD_frame.png
├─ ui/        界面元素     UI_<element>[_<state>].png
├─ audio/     音频         BGM_<name>.ogg / SFX_<name>.wav
└─ fx/        特效叠层     FX_<name>.png
metadata/     NFT JSON     <token>.json（指向 IPFS 图）
deliverables/ 提交用高清    HERO_<name>_1920x1080.png（SuperZygo 效果图）
```

- 全小写 + 下划线；变体用后缀（`_legendary` / `_warm`）。
- 命名与内容/链上文档的 pieceId、场景名一一对应。

|---|---|---|

| 辅·陶褐 | ▨ | `#8A5A3B` |
| 亮·米白 | ▨ | `#F3E7D0` |
| 冬·冷白 | ▨ | `#C7D2D6` |
| 冬·青灰 | ▨ | `#8FA3AB` |
| 暗·墨 | ▨ | `#2A211C` |
| 点缀·金 | ▨ | `#E8C06A` |
| 强调·青花蓝 | ▨ | `#3E6B8C` |

> 全项目锁定此调色板，AI 生成后统一量化到这套色，保证一致性。青花蓝仅用于青花酒盏与图鉴强调。

### 2.2 像素规格

| 类别 | 作图分辨率 | 交付分辨率 | 说明 |
|---|---|---|---|
| 场景背景 BG | 640×360 | 640×360 PNG | 16:9，CSS `image-rendering: pixelated` 放大 |
| 人物立绘 CH | 220×320 | 220×320 透明 PNG | 半身，置于对话侧 |
| 信物/徽章 IT | 512×512 | 1024×1024 透明 PNG | NFT 展示图；另出高清见 §11 |
| 文化卡插画 | 320×200 | 320×200 | 嵌入卡框 360×520 |
| UI 元素 | 按需 | 透明 PNG | 对话框用九宫格 |

### 2.3 风格基准（Style Bible）

- 中低像素密度、清晰轮廓、暖光；不追写实，追"温润的记忆感"。
- 统一光源偏暖、右上方；阴影用陶褐/墨而非纯黑。
- **AI 提示词前缀模板**（所有素材共用，保证同源）：
  > `pixel art, warm nostalgic mood, limited palette (amber red, warm wood, cream, cold winter white), soft warm lighting, clean outlines, cozy, Chinese Fujian folk aesthetic, <具体内容>, no text`
- 一致性做法：固定参考图 + i2i、固定 seed、生成后统一降采样 + 调色板量化（见 §12）。

## 12. AI 生成流程与一致性策略

1. **定基准**：先出 1 张场景 + 1 个信物作"风格锚"，全员据此对齐。
2. **锁调色板**：生成后统一降采样（如缩到目标分辨率）+ 量化到 §2.1 调色板（工具：aseprite / ImageMagick `-remap palette.png` / Python PIL）。
3. **同源**：固定参考图 + image-to-image + 固定 seed，保证人物/场景跨图一致。
4. **透明底**：人物/信物出图后抠背景（或直接生成纯色底再抠）。
5. **像素化**：非像素模型出图 → 降采样 + 边缘清理 → 得到干净像素。
6. **批处理脚本**：写一个 `tools/pixelize` 批量降采样+调色板量化，统一风格。

## 13. 交付格式与验收标准

- 格式：PNG（透明用 32 位）；音频 OGG/WAV；文本 JSON/YAML。
- 尺寸严格按 §2.2；命名严格按 §1。
- 验收：①在目标调色板内 ②边缘干净无杂色 ③放大后无糊 ④与风格锚一致 ⑤信物另有 ≥1920×1080 高清版。
- 全部素材注明来源与授权（尤其库素材），汇总进 `CREDITS.md`。
